Everything you need to know

Yearly Discoveries is a tool that helps you explore the songs you discovered in a specific year. It filters your Spotify playlists to show you songs that were actually released in that year—not songs that just became popular in that year.

For example, if a song was released in 2020 but received heavy airplay in 2023, it won't appear in your 2023 discoveries—it will appear in 2020.

Many "Best of 202X" lists highlight songs that were popular in that year, but not necessarily released in that year. This tool solves that problem by showing you which songs you discovered that were actually from a specific year.

It's great for creating annual playlists that truly represent the music released in that year, or for seeing how your musical taste evolved over time.

Yearly Discoveries has two modes:

🌐 Web Mode

Browse your yearly discoveries interactively in your browser. Filter by year, sort songs different ways, and explore without modifying your playlists.

⚙️ Batch Mode

Automatically scan your playlists and populate a target playlist with songs from a specific year. This runs on a schedule or manually via CLI.

We use Spotify's API to check the release date of each song. If a song's release date falls within the year you're browsing, it appears in your discoveries for that year.

For example:

  • Song released January 15, 2023 → Appears in 2023
  • Song released December 31, 2023 → Appears in 2023
  • Song released January 1, 2024 → Appears in 2024

We only use the release year, not the specific date within that year.

The web app shows all songs from playlists in your account that match your filter settings. By default, it shows songs you've marked as favorites (❤️ in Spotify).

However, you can configure the batch mode to include all songs you've added, regardless of whether they're in your library.

The actual filtering depends on how you've configured batch mode—web mode respects those same settings when pulling data.

In web mode, the app accesses:

  • Your playlists and the songs in them
  • Metadata about those songs (title, artist, release date, album art)
  • Your library status for those songs

Data storage: The web app stores session information in memory only—nothing is saved to a database. Once you log out or close the app, your session is cleared.

Spotify playlists: In batch mode, the app can modify your playlists, but only if you explicitly configure it to do so.

To show you your yearly discoveries, we need permission to:

  • Read your playlists — To find songs you've added
  • Read your library — To see which songs you've marked as favorites
  • Modify playlists — Only used in batch mode, if configured

These permissions are requested by Spotify, not by us. We only use them to provide the functionality you've configured.

Batch mode runs from the command line and automatically populates a target playlist with your yearly discoveries. To use it:

  1. Create a Spotify application and get your client ID and secret
  2. Create an empty playlist in Spotify to hold your discoveries
  3. Configure environment variables (see the README on GitHub)
  4. Run make token to get an OAuth token
  5. Run make run to populate your playlist

For detailed instructions, see the project README.

Here are some common reasons:

  • Song is from a different year — Check Spotify's release date for the song. It might be from a different year than you thought.
  • Song isn't in your library — By default, we only show songs you've marked as favorites (❤️). If you've added the song to playlists but not your library, adjust your settings in batch mode.
  • Song isn't in a playlist — The web app only shows songs from playlists in your account. Make sure the song is actually in one of your playlists.
  • Release date ambiguity — Some songs have different release dates for the album vs. the single. Spotify uses the earliest release date.

Yearly Discoveries is built with:

  • Go — The backend server and CLI tool
  • Spotify Web API — For accessing your playlists and songs
  • OAuth 2.0 — For secure authentication
  • Tailwind CSS — For the web UI styling
  • JavaScript — For frontend interactivity
